📚 What is MABIMS?

The Islamic calendar operates on a lunar cycle, while the Gregorian calendar is solar-based, leading to discrepancies in dates. For region-specific accuracy, the Mabims standard (an abbreviation for a commonly used variant) is critical. This standard is widely adopted by governing bodies in countries like Singapore, Indonesia, and Malaysia.

The Need for Specialization

While many general-purpose APIs default to using the Ummul Qura Hijri calendar (used predominantly by Saudi Arabia), this standard may not reflect local governmental calendars. The Mabims API was specifically developed to address these regional variations, providing accurate and localized conversions compliant with key Southeast Asian standards.

🔬 API Methodology & Scope

Source Calendar: For the highest level of local accuracy, this API uses the Indonesian Kemenag (Ministry of Religious Affairs) calendar as its primary source reference.

Current Limitation: Due to the complexity and ongoing validation required for regional standards, this version is currently validated and guaranteed accurate only for dates within the years 2025 and 2026. For future date ranges or advanced calendar features, please contact us.

⚙️ API Reference: The Lookup Endpoint

Use our single endpoint, `/lookup`, to perform all conversions. All input must be date strings in the format YYYY-MM-DD.

GET mabims-api.netlify.app/lookup

Parameters

Parameter Type Required? Description
date String (YYYY-MM-DD) Yes The date to be converted.
source String (enum) Yes Specifies the input format: gregorian or hijri.

💡 Example Usage

To convert the Gregorian date January 3rd, 2025, to Hijri:

GET /lookup?date=2025-01-03&source=gregorian

Response Body (JSON Schema)

✅ Success Response (HTTP 200)

{ "success": true, "input_data": { "date": "YYYY-MM-DD", "type": "SOURCE_TYPE" }, "output_data": { "date": "Output Date String", "type": "TARGET_TYPE" }, "message": "Conversion successful." }

❌ Error Response (HTTP 400/404)

{ "success": false, "input_data": { "date": "...", "type": "..." }, "error": "Data not found.", "message": "No calendar pair exists for the provided date and source." }
